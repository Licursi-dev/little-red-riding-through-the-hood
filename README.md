Little Red Riding Through the Hood

A retro-inspired browser-based interactive story game built with Python, FastAPI and custom game logic.

Overview

Little Red Riding Through the Hood is a choice-based interactive story game inspired by classic text adventures. Players take the role of Little Red, a high school student navigating the streets of Los Angeles in 1993, where rival gangs, school bullies and difficult choices can turn an ordinary day into a dangerous one.

The game combines branching story paths, inventory management, multiple endings and a lives system, all presented through a retro-inspired web interface designed to resemble classic arcade and early computer games.

Project Inspiration

This project began while I was studying for my IT Diploma at college.

Late in the course, a Python module was introduced and students were asked to complete a small Python assignment. Many students chose to follow a basic example provided during the lesson, but I wanted to challenge myself by building something original.

A classmate, James Winnard, shared the same enthusiasm. We both enjoyed classic text-based adventure games and spent an evening discussing ideas over text messages. Together we designed and built the original version of Little Red Riding Through the Hood in a single evening, working for approximately ten hours to create a complete playable experience before the assignment deadline.

The project was successfully submitted and passed as part of the course.

After college had finished, I revisited the project while considering how to develop my portfolio. Returning to the code allowed me to identify design limitations, improve the story structure and resolve a number of issues that existed in the original version.

The current version represents a complete redevelopment of the original concept. The game engine was rewritten, additional story paths and endings were added, the user interface was redesigned with custom CSS, and FastAPI was introduced to transform the project from a simple Python assignment into a browser-based game that others could play and enjoy.

Although the second version was developed independently, I still consider the project a team effort. James played an important role during the original development stage by helping turn an idea into a completed project. The experience reinforced an important lesson that applies throughout software development: teamwork, encouragement and shared enthusiasm often provide the momentum needed to take projects from concept to completion.

This project helped strengthen my understanding of:
